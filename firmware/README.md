# Обновление прошивки по воздуху (OTA с GitHub)

Контроллер периодически (и по кнопке в веб-интерфейсе) читает `firmware/manifest.json`
из ветки `main` этого репозитория по HTTPS, сравнивает `version` с текущей прошивкой и,
если в манифесте версия новее, скачивает `url` (образ из GitHub Releases) и обновляется.
При неудачном старте новой прошивки — автоматический откат на предыдущую.

## Как выпустить обновление

1. Поднять версию в корневом `CMakeLists.txt`:
   ```
   set(PROJECT_VER "0.4.0")
   ```
2. Собрать: `idf.py build` → образ в `build/brewcontrol.bin`.
3. Создать GitHub Release с тегом `v0.4.0` и приложить `brewcontrol.bin`:
   ```
   gh release create v0.4.0 build/brewcontrol.bin -t "v0.4.0" -n "описание"
   ```
4. Обновить `firmware/manifest.json` (version и url на новый тег) и запушить в `main`.

В течение часа (или сразу по кнопке «Проверить обновление») все контроллеры с доступом
в интернет подтянут новую прошивку сами.

## Формат манифеста

```json
{
  "version": "0.4.0",
  "url": "https://github.com/pharmsoft/brew-control/releases/download/v0.4.0/brewcontrol.bin",
  "notes": "краткое описание"
}
```

Сравнение версий — по числовым компонентам `major.minor.patch`; обновление применяется,
только если версия в манифесте строго новее установленной (downgrade не выполняется).
